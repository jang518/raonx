# DESIGN.md — 경영현황 대시보드 디자인 기준

> 목적: Codex가 경영현황 대시보드, 보고서, 로그인 화면, PDF/인쇄용 출력물을 만들 때 참고할 라온엑스 디자인 기준입니다.
> 이 문서는 예쁜 화면을 만들기 위한 문서가 아니라, 경영진과 경영지원팀이 숫자를 빠르게 이해하고 회의에서 바로 사용할 수 있도록 만드는 기준 문서입니다.

## 1. 브랜드/서비스 인상

이 대시보드는 라온엑스 경영팀이 AI를 현업에 적용하기 위한 사내 경영현황 대시보드다.

핵심 이미지는 다음과 같다.

* 현대적인 SaaS 대시보드
* 비전문가도 이해할 수 있는 경영현황 화면
* 경영회의에서 바로 공유 가능한 화면
* 숫자와 근거가 명확한 보고서형 화면
* 과하게 화려하지 않고 신뢰감 있는 업무용 화면

라온엑스 CI는 그레이 계열의 안정감 있는 로고와 청록색 포인트 `X`를 사용한다. 대시보드 역시 이 방향을 따른다. 전체 화면은 밝고 깔끔하게 구성하되, 핵심 수치와 경고는 명확하게 보이게 한다.

## 2. 디자인 키워드

* 현대적: SaaS 대시보드처럼 깔끔하고 정돈된 인터페이스
* 신뢰감: 숫자, 기준 기간, 목표 기준을 명확하게 표시
* 쉬운 이해: 비전문가도 카드와 차트만 보고 의미를 알 수 있게 구성
* 회의 친화적: 경영회의에서 화면을 띄워 바로 설명할 수 있게 구성
* 절제된 강조: 중요한 수치는 강조하지만 색상과 장식은 과하지 않게 사용
* 데이터 중심: 감성 문구보다 매출, 이익, 경비, 목표 달성률 중심

## 3. 색상 토큰

### 기본 색

* `canvas` — #F6F8FA: 기본 배경. 밝고 중립적인 업무용 배경.
* `surface` — #FFFFFF: 카드, 표, 대시보드 패널.
* `surface-soft` — #F1F4F5: 보조 패널, 필터 영역, 로그인 박스 배경.
* `ink` — #1F2426: 제목과 핵심 텍스트.
* `body` — #3F4749: 본문 텍스트.
* `muted` — #6E7779: 보조 설명, 캡션, 날짜.
* `hairline` — #DDE3E5: 카드 경계, 표 구분선.
* `disabled` — #B8C0C2: 비활성 텍스트, 비활성 버튼.

### 브랜드 포인트

* `raonx-gray-dark` — #5B6261: 라온엑스 로고의 진한 그레이 계열. 제목, 보조 강조, 네비게이션에 사용.
* `raonx-gray-light` — #B8BEC0: 라온엑스 로고의 밝은 그레이 계열. 보조선, 배경 강조, 비활성 상태에 사용.
* `raonx-cyan` — #36BFC7: 라온엑스 CI의 청록 포인트. 주요 강조, 선택 상태, 핵심 액션에 사용.
* `raonx-cyan-soft` — #E6F8FA: 청록색의 약한 배경. 선택된 필터, 정보 배지에 사용.
* `navy-ink` — #263238: 전문성과 안정감을 주는 진한 텍스트/헤더 색상.

### 데이터 시각화 색

* 매출: `raonx-cyan` #36BFC7
* 매출이익: #4F8F9A
* 영업이익: #2F6F7E
* 경상이익: #5661A9
* 경비: #8A9497
* 목표 달성률: #20A67A
* 목표대비부족액: #D97706
* 경고/위험: #DC2626
* 정보/참고: #64748B
* 비활성/데이터 없음: #CBD5D9

### 경고 색상 원칙

경고와 위험 신호는 빨간색으로 명확히 표시한다. 단, 빨간색을 과다 사용하지 않는다.

* 치명적 오류: #DC2626
* 주의 필요: #D97706
* 참고/확인 필요: #64748B
* 정상/양호: #20A67A

빨간색은 다음 상황에만 사용한다.

* 데이터 오류
* 목표 대비 심각한 부족
* 음수 또는 비정상 수치
* 로그인 실패
* 운영상 즉시 확인이 필요한 상태

## 4. 타이포그래피

* 기본 폰트: Pretendard, Apple SD Gothic Neo, Noto Sans KR, sans-serif
* 숫자 폰트: Inter, Pretendard, sans-serif
* 제목: 28–34px, 700, 자간 -0.02em
* 섹션 제목: 20–24px, 700
* 카드 제목: 14–15px, 600
* 본문: 14–16px, 400–500, line-height 1.55
* 캡션/메타: 12–13px, 400, `muted`
* KPI 숫자: 32–44px, 700, line-height 1.1
* 표 숫자: 13–14px, 500, 우측 정렬
* 버튼: 14px, 600

문장은 짧고 명확하게 쓴다. 감성적인 표현보다 경영회의에서 바로 말할 수 있는 문장을 우선한다.

좋은 문장:

* “전월 대비 영업이익이 감소했습니다.”
* “목표 달성률이 낮은 팀을 확인하세요.”
* “월 목표가 없어 분기 목표 기준 참고값으로 표시합니다.”

피해야 할 문장:

* “상황이 좋아 보입니다.”
* “앞으로 매출이 증가할 것 같습니다.”
* “이 팀은 실적이 나쁩니다.”

## 5. 레이아웃 원칙

대시보드는 개인 PC 브라우저와 노트북 화면을 우선한다.

* 기본 화면 폭은 1280px 이상 데스크톱 브라우저를 기준으로 한다.
* 12컬럼 그리드 또는 카드 기반 반응형 레이아웃을 사용한다.
* 첫 화면 상단에는 핵심 KPI 카드를 배치한다.
* KPI 카드는 한 줄에 4개를 기본으로 하고, 화면 폭에 따라 3개 또는 2개로 줄인다.
* KPI가 6개 이상이면 2줄 배치 또는 중요도에 따라 상단/보조 KPI로 나눈다.
* 필터는 상단 또는 좌측에 고정한다.
* 차트와 표는 한 화면에 너무 많이 몰아넣지 않는다.
* 카드 간격은 16–24px, 섹션 간격은 32–48px을 기본으로 한다.
* 모바일은 필수 우선순위가 아니지만, 깨지지 않는 반응형은 유지한다.

첫 화면 권장 순서:

1. 대시보드 제목 + 기준 연도/기간 + 마지막 업데이트 시간
2. 전역 필터
3. 핵심 KPI 카드
4. 경영회의용 한 줄 요약 카드
5. 주요 추이 차트
6. 팀별 비교 차트
7. 목표 달성률/부족액 카드
8. 팀별 상세 테이블
9. 데이터 품질 경고

## 6. 컴포넌트 기준

### 카드

* 배경: `surface`
* 테두리: 1px `hairline`
* 라운드: 16px
* 그림자: 약하게 사용

  * `0 8px 24px rgba(15, 23, 42, 0.06)` 이하
* 카드 내부 여백: 20–24px
* 카드 제목과 숫자는 명확히 분리
* 카드마다 가능한 경우 한 줄 요약을 포함

KPI 카드 구조:

1. 지표명
2. 핵심 숫자
3. 전월 대비 또는 목표 대비 변화
4. 한 줄 해석
5. 데이터 기준 또는 보조 설명

### 버튼

* 기본 CTA: `raonx-cyan` 배경 + 흰색 텍스트
* 보조 버튼: 흰색 배경 + `raonx-gray-dark` 텍스트 + 1px `hairline`
* 위험 버튼: 빨간색 계열. 삭제/초기화/오류 작업에만 사용
* 라운드: 10–12px
* 버튼 문구는 짧고 동작 중심으로 쓴다.

예:

* “조회”
* “필터 적용”
* “PDF 출력”
* “데이터 새로고침”
* “로그인”

### 배지

* 정상: 초록 계열 배경 + 초록 텍스트
* 주의: 주황 계열 배경 + 주황 텍스트
* 위험: 빨강 계열 배경 + 빨강 텍스트
* 정보: 청록 또는 회색 계열 배경 + 정보 텍스트
* 데이터 없음: 회색 배경 + 회색 텍스트

예:

* `목표 초과`
* `목표 미달`
* `월 목표 없음`
* `분기 기준 참고`
* `데이터 확인 필요`
* `FCST 미적용`

### 필터

필터는 비전문가도 이해하기 쉽게 구성한다.

기본 필터 후보:

* 연도
* 기간 유형: 월 / 분기 / 반기 / 연간
* 월
* 분기
* 팀
* 지표
* 목표 기준

필터 선택 상태는 `raonx-cyan-soft` 배경 또는 `raonx-cyan` 테두리로 표시한다.

### 테이블

* 숫자는 우측 정렬
* 팀명과 항목명은 좌측 정렬
* 헤더는 고정 가능하면 고정
* 합계 행은 강조
* 목표 미달, 음수, 오류 값은 배지 또는 텍스트 색으로 표시
* 너무 많은 컬럼을 한 번에 보여주지 않는다
* 상세 테이블은 정렬 가능하게 만든다

## 7. 차트 기준

차트는 데이터에 따라 적합한 스타일을 선택한다.

권장 차트:

| 목적         | 권장 차트             |
| ---------- | ----------------- |
| 월별 매출 추이   | 라인 차트 또는 막대+라인 혼합 |
| 영업이익 추이    | 라인 차트             |
| 매출이익 추이    | 라인 차트             |
| 팀별 매출 비교   | 가로 막대 차트          |
| 팀별 매출이익 비교 | 가로 막대 차트          |
| 팀별 영업이익 비교 | 가로 막대 차트          |
| 팀별 매출 기여도  | 누적 막대 또는 도넛 차트    |
| 목표 달성률     | 진행률 바 또는 게이지형 카드  |
| 목표대비부족액    | 막대 차트 또는 경고 카드    |
| 전월 대비 증감률  | 증감 카드 또는 워터폴 후보   |

차트 원칙:

* 한 차트에 너무 많은 색을 쓰지 않는다.
* 주요 지표 색은 지표별로 고정한다.
* 팀별 비교는 팀별 고정 색상을 사용하되, 너무 화려하지 않게 한다.
* 범례는 차트 아래 또는 오른쪽에 작게 배치한다.
* Y축 단위와 금액 단위를 명시한다.
* 0, 없음, 오류는 구분해서 표시한다.
* 차트 제목은 단순형, 부제는 해석형으로 작성한다.

예:

* 제목: `영업이익 추이`
* 부제: `전월 대비 감소한 구간은 경비 증가 여부를 함께 확인하세요.`

## 8. 팀별 색상 기준

팀별 비교에서는 팀마다 고정 색상을 부여한다. 단, 라온엑스 브랜드 색과 충돌하지 않도록 채도는 낮추고, 주요 강조색 `raonx-cyan`은 핵심 지표 또는 선택 상태에 우선 사용한다.

초기 추천 색상:

| 팀     | 색상      | 용도             |
| ----- | ------- | -------------- |
| SDT   | #3B82F6 | 개발팀            |
| TST1  | #14B8A6 | 국내 영업팀         |
| TST2  | #06B6D4 | 해외 영업팀         |
| MKT   | #A855F7 | 마케팅팀           |
| EST1  | #22C55E | 기술 용역 및 기술 지원팀 |
| EST2  | #F59E0B | 기술 용역팀         |
| EST3  | #64748B | 기술 지원팀         |
| 경영지원팀 | #475569 | 경영지원팀          |

팀별 색상은 `config/dashboard.yml` 또는 별도 설정 파일에서 바꿀 수 있게 한다. 코드에 직접 하드코딩하지 않는다.

## 9. 금액 및 숫자 표기

금액 단위는 자동 변환을 기본으로 한다.

권장 표기:

| 금액 범위               | 표시 방식                 |
| ------------------- | --------------------- |
| 1,000만원 미만          | `만원` 또는 `원`           |
| 1,000만원 이상 ~ 1억원 미만 | `천만원` 또는 `백만원`        |
| 1억원 이상              | `억원`                  |
| 표 상세                | `백만원` 또는 `원` 기준 고정 가능 |

초기 권장:

* KPI 카드: 자동 변환

  * 예: `12.3억`, `8,400만`
* 표: `백만원` 기준 또는 사용자가 선택한 단위
* 툴팁: 원 단위 전체값 표시
* 증감률: `+12.3%`, `-4.5%`
* 음수: `-1.2억`
* 데이터 없음: `-`
* 계산 불가: `계산 불가`
* 목표 없음: `목표 없음`

숫자 원칙:

* 천 단위 구분 쉼표 사용
* 증감률은 소수점 1자리 기본
* 금액은 화면 성격에 따라 자동 축약
* 기준 단위는 카드 또는 차트에 반드시 표시

## 10. 로그인 화면 기준

로그인 화면은 단순하게 만든다.

구성:

* 라온엑스 로고
* 서비스명: `경영현황 대시보드`
* 아이디 입력
* 비밀번호 입력
* 로그인 버튼
* 오류 메시지
* 내부 시스템 안내 문구

스타일:

* 밝은 배경
* 중앙 정렬 카드
* 과한 애니메이션 금지
* 라온엑스 청록색을 로그인 버튼에 사용
* 로그인 실패는 빨간색으로 명확히 표시하되, 상세 보안 정보는 노출하지 않음

안내 문구 예:

> 라온엑스 내부 사용자 전용 시스템입니다.

## 11. PDF/인쇄 출력 기준

인쇄/PDF 출력을 고려한다.

원칙:

* A4 가로 방향을 우선 고려
* 배경색은 너무 진하지 않게
* 카드 그림자는 인쇄 시 최소화
* 표와 차트가 잘리지 않게 페이지 분리 고려
* 로고, 기준 기간, 출력일을 상단에 표시
* PDF에서는 인터랙션 요소를 제거하고 정적인 설명을 추가
* 경고/주의 색상은 흑백 출력에서도 구분 가능하게 아이콘 또는 라벨을 함께 사용

PDF용 구성 후보:

1. 표지/제목
2. 핵심 KPI 요약
3. 목표 달성률 요약
4. 팀별 실적 비교
5. 전월 대비 변화
6. 데이터 품질 경고
7. 해석상 주의사항

## 12. 대시보드 출력물 기준

대시보드는 예쁘기보다 의사결정이 빨라야 한다.

첫 화면에서 보여줄 것:

1. 총매출
2. 매출이익
3. 영업이익
4. 경비
5. 목표 달성률
6. 전월 대비 영업이익 증감률
7. 전월 대비 경비 증감률
8. 팀별 실적 비교
9. 목표 대비 부족액
10. 데이터 누락/오류/목표 없음 경고

모든 결론에는 가능한 한 “근거 데이터”와 “다음 확인 항목”이 같이 있어야 한다.

카드 한 줄 요약 예:

* `목표 달성률은 양호하지만 경비 증가율을 함께 확인하세요.`
* `월 목표가 없어 분기 목표 기준 참고값으로 표시합니다.`
* `기술지원팀은 단순 매출보다 지원 기여도를 함께 봐야 합니다.`
* `전월 대비 영업이익이 감소했습니다. 경비 증가 항목을 확인하세요.`

## 13. 금지 사항

* 빨간색을 과다 사용하지 않는다.
* 숫자 근거 없는 경영 판단을 쓰지 않는다.
* 목표 기준이 불명확한데 달성률을 확정적으로 표시하지 않는다.
* 월 목표가 없는데 월 목표 달성률처럼 보이게 하지 않는다.
* `실적G`가 없는데 FCST가 있는 것처럼 표시하지 않는다.
* 데이터 없음, 계산 불가, 목표 없음 상태를 숨기지 않는다.
* 차트를 많이 넣는 대신 핵심 판단이 안 보이게 만들지 않는다.
* 실제 고객사명, 계약 금액, 담당자 정보가 불필요하게 화면에 노출되지 않게 한다.
* 과한 애니메이션, 과한 그라데이션, 장식용 그래픽을 사용하지 않는다.
* 로고 색상과 충돌하는 형광색 계열을 남발하지 않는다.
* 경영회의용 화면에 너무 작은 글씨를 사용하지 않는다.
