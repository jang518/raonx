# DESIGN.md — 라온엑스 고객 니즈-기능 매칭 대시보드 디자인 기준

> 목적: Codex가 `knowledge_for_marketing_dashboard`의 Streamlit 대시보드, 조회 화면, 상태 표시, 권한 안내, 향후 PDF/인쇄용 출력물을 만들 때 참고할 디자인 기준입니다. 이 문서는 단순히 예쁜 화면을 만들기 위한 문서가 아니라, 마케팅 자료 기획과 고객 미팅 준비에 필요한 고객 니즈, 추천 소프트웨어 기능, 제안 메시지, 다음 확인 질문을 빠르게 이해하고 바로 활용할 수 있도록 만드는 기준 문서입니다.

---

## 1. 디자인 목표

이 대시보드는 라온엑스 내부에서 고객 방문보고서와 소프트웨어 기능 정보를 연결해 활용하기 위한 **사내망 내부 전용 지식 검색형 대시보드**입니다.

디자인 목표는 크게 두 가지입니다.

1. **마케팅 활용**
   - 고객 니즈와 소프트웨어 기능 정보를 바탕으로 세미나, 제안서, 고객 맞춤 기능 소개, 기술 콘텐츠 기획에 활용할 수 있게 한다.
   - 반복적으로 등장하는 니즈, 소프트웨어별 가치 포인트, 고객군별 관심 주제를 빠르게 파악할 수 있게 한다.
   - 단순 기능 나열이 아니라 “어떤 고객 문제에 어떤 가치 메시지를 연결할 수 있는가”가 보이게 한다.

2. **고객 미팅 준비**
   - 고객 미팅 전 3~5분 안에 핵심 정보를 파악한다.
   - 고객 니즈 → 추천 기능 → 제안 가치 → 다음 미팅 질문 흐름이 한눈에 보이게 한다.
   - 낮은 confidence, 데이터 누락, 확인 필요 항목을 숨기지 않고 명확히 표시한다.

공통 목표:

- 영업 담당자, 엔지니어, 마케팅 담당자가 모두 이해할 수 있는 업무형 문구를 사용한다.
- 내부 업무 시스템처럼 신뢰감 있고 절제된 화면을 만든다.
- 마케팅 활용과 고객 미팅 준비 모두에서 근거 없는 과장 표현을 피한다.

## 2. 설계 원칙

| 원칙 | 설명 |
|---|---|
| 마케팅 활용 지원 | 반복 니즈, 가치 메시지, 소프트웨어별 강점, 고객군별 관심 주제를 콘텐츠·제안 기획에 활용할 수 있어야 한다. |
| 미팅 준비 지원 | 모든 화면은 “고객 미팅 전에 무엇을 확인해야 하는가?”에 답해야 한다. |
| 근거 기반 | 고객 니즈, 추천 기능, 제안 메시지는 방문보고서, wiki, export CSV에 근거해야 한다. |
| confidence 중심 | 니즈-기능 매칭의 신뢰도는 현재 CSV에 존재하는 `confidence`를 기준으로 표현한다. |
| 상태 노출 | 낮은 confidence, 데이터 누락, 확인 필요 항목은 숨기지 않는다. |
| 민감도 미사용 | 내부 직원용 대시보드이므로 `sensitivity_level`은 표시/필터/배지 기준으로 사용하지 않는다. |
| 노출 기준 준수 | `dashboard_visible=FALSE`는 기본 화면에서 숨긴다. |
| 단정 금지 | 구매 가능성, 예산, 도입 의지, 고객 내부 의사결정 상황을 추정하지 않는다. |
| 권한 분리 | admin은 수정 가능, viewer는 조회 전용으로 명확히 구분한다. |
| 확장 가능 | 소프트웨어명은 하드코딩하지 않고 CSV의 `software` 값을 기준으로 처리한다. |

## 3. 브랜드와 로고 기준

라온엑스 CI는 그레이 계열의 안정감 있는 로고와 청록색 포인트 `X`를 사용합니다. 대시보드도 이 방향을 따릅니다.

로고 사용 원칙:

- 라온엑스 로고는 실제 제공된 PNG 파일을 우선 사용한다.
- 로고 파일이 없으면 외부 URL에서 임의로 가져오지 않고 텍스트 로고 또는 임시 placeholder로 처리한다.
- 로고 원본 색상을 변경하지 않는다.
- 로고를 찌그러뜨리거나 과도하게 확대하지 않는다.
- 모든 카드마다 로고를 반복하지 않는다.

| `disabled` | `#B8C0C2` | 비활성 텍스트, 비활성 버튼 |

| `raonx-gray-light` | `#B8BEC0` | 보조선, 배경 강조, 비활성 상태 |

| `navy-ink` | `#263238` | 안정감 있는 헤더/텍스트 |

### 4.3 상태 색

| 상태 | 색상 | 용도 |
|---|---|---|
| confidence 높음 | `#20A67A` | 신뢰도 높음 |
| confidence 중간 / 확인 필요 | `#D97706` | 중간 confidence, 확인 필요 |
| confidence 낮음 / 오류 | `#DC2626` | 낮은 confidence, CSV 오류, 로그인 실패 |
| 정보/참고 | `#64748B` | 참고, 보조 정보 |
| 데이터 없음 | `#CBD5D9` | 결측, 데이터 없음 |

주의:

- 빨간색은 낮은 confidence, 로그인 실패, CSV 오류처럼 즉시 주의가 필요한 상태에만 사용한다.
- `sensitivity_level`은 색상·배지 체계에 사용하지 않는다.

## 5. 타이포그래피와 문장 원칙

기본 폰트:

```text
Pretendard, Apple SD Gothic Neo, Noto Sans KR, sans-serif
```

문장 원칙:

- 짧고 명확하게 쓴다.
- 기능 설명보다 업무 판단에 도움이 되는 표현을 우선한다.
- 고객이 이미 답한 것처럼 표현하지 않는다.
- 데이터에 없는 고객 의도나 구매 가능성을 쓰지 않는다.
- 마케팅 활용 후보를 확정 캠페인처럼 쓰지 않는다.

좋은 문장:

- “이 고객 니즈와 연결된 추천 기능을 확인하세요.”
- “다음 미팅에서 확인할 질문을 검토하세요.”
- “confidence가 낮은 매칭은 담당자 확인 후 제안에 활용하세요.”
- “반복적으로 등장하는 니즈를 마케팅 콘텐츠 후보로 검토하세요.”

피해야 할 문장:

- “이 고객은 구매 가능성이 높습니다.”
- “이 기능을 제안하면 수주될 것 같습니다.”
- “고객이 반드시 이 기능을 원합니다.”
- “예산이 충분할 것으로 보입니다.”
- “이 마케팅 캠페인은 확정입니다.”

## 6. 레이아웃 원칙

- 기본 화면 폭은 1280px 이상 데스크톱 브라우저를 기준으로 한다.
- 첫 화면은 `Home`을 가장 우선한다.
- 첫 화면 상단에는 라온엑스 로고, 서비스명, 로그인 사용자/권한, 마지막 데이터 업데이트 시간을 표시한다.
- 검색창은 사용자가 가장 먼저 보게 상단에 배치한다.
- 필터는 처음부터 과도하게 만들지 않고, 1차 개발에서는 일반 검색창 중심으로 구성한다.
- 고객사명과 담당자는 표시하되, 불필요하게 큰 시각 강조를 하지 않는다.
- admin 수정 UI는 viewer에게 보이지 않게 한다.
- `sensitivity_level` 관련 필터나 배지는 만들지 않는다.

Home 권장 순서:

1. 라온엑스 로고 + 대시보드 제목 + 로그인 사용자/권한 + 마지막 업데이트 시간
2. 전역 검색창
3. 핵심 요약 카드
4. 최근 고객 니즈
5. 자주 등장한 니즈 키워드
6. 낮은 confidence 또는 확인 필요 매칭 현황
7. 마케팅 활용 후보 요약
8. 고객사별 대응 전략 바로가기
9. 소프트웨어별 가치 포인트 바로가기
10. 데이터 품질 경고

## 7. 핵심 컴포넌트 기준

### 7.1 카드

고객 니즈 카드 구조:

1. 고객사명 또는 고객 ID
2. 산업군
3. 니즈 키워드
4. 니즈 요약 또는 최종 정리
5. 니즈 날짜
6. 담당자
7. 연결된 추천 기능 개수

기능 매칭 카드 구조:

1. 니즈 키워드
2. 소프트웨어명
3. 기능명
4. 가치 메시지
5. confidence 배지
6. 다음 확인 질문 또는 참고 안내

고객사별 대응 전략 카드 구조:

1. 고객사명 또는 고객 ID
2. 주요 니즈
3. 추천 소프트웨어
4. 접근 방향
5. 제안 메시지
6. 다음 미팅 추천 질문
7. admin override 메모가 있는 경우 별도 표시

마케팅 활용 카드 구조:

1. 반복 니즈 키워드
2. 관련 산업군 또는 고객군
3. 연결 가능한 소프트웨어/기능
4. 가치 메시지 후보
5. 콘텐츠 또는 세미나 주제 후보
6. 근거 데이터 또는 출처
7. 활용 전 확인 사항

### 7.2 배지

| 데이터 | 표시 |
|---|---|
| `confidence=높음` | 초록 계열 배지 |
| `confidence=중간` | 주황 계열 배지 |
| `confidence=낮음` | 빨강 계열 배지 |
| 데이터 없음 | 회색 배지 |
| `dashboard_visible=FALSE` | 기본 화면에서는 표시하지 않음 |

### 7.3 필터

1차 개발 기본 필터 후보:

- 전역 검색창
- 고객사명 또는 고객 ID
- 산업군
- 니즈 키워드
- 소프트웨어
- confidence
- dashboard_visible
- 화면 목적: 마케팅 활용 / 고객 미팅 준비

### 7.4 테이블

- 고객사명, 니즈 키워드, 소프트웨어명은 좌측 정렬한다.
- ID, 날짜, 상태값은 중앙 또는 좌측 정렬한다.
- 긴 텍스트는 기본적으로 2~3줄까지만 보이고 상세 보기에서 전체 표시한다.
- `confidence`는 텍스트만 쓰지 말고 배지로 표시한다.
- 현재 CSV에는 `review_status`가 없으므로, review_status UI는 만들지 않는다.
- `sensitivity_level`은 표시/필터/배지 기준으로 사용하지 않는다.

## 8. 화면별 디자인 기준

### 8.1 Home

첫 화면에서 보여줄 것:

1. 라온엑스 로고와 서비스명
2. 로그인 사용자와 권한
3. 마지막 데이터 업데이트 시간
4. 전체 고객 니즈 수
5. 전체 매칭 수
6. confidence 낮음 항목 수
7. 최근 고객 니즈
8. 자주 등장한 니즈 키워드
9. 마케팅 활용 후보 니즈 또는 가치 메시지
10. 데이터 품질 경고
11. 고객사별 대응 전략 바로가기

### 8.2 고객 니즈 검색

- 고객사명, 고객 ID, 산업군, 니즈 키워드, 니즈 요약을 함께 검색할 수 있게 한다.
- `dashboard_visible=FALSE` 또는 이에 준하는 항목은 기본 결과에서 숨긴다.

### 8.3 니즈-기능-가치 매칭

- 고객 니즈 → 추천 소프트웨어 → 기능 → 가치 메시지 순서로 읽히게 한다.
- `confidence`를 반드시 배지로 표시한다.
- 낮은 confidence 항목은 확정 추천처럼 보이지 않게 한다.
- 가치 메시지 아래에 “다음 확인 질문” 또는 “활용 전 확인 사항”을 함께 표시한다.

### 8.4 고객사별 대응 전략

- 고객사명 또는 customer_id 기준으로 검색한다.
- `main_needs`, `recommended_software`, `approach_details`, `suggested_questions`, `proposal_message`를 순서대로 보여준다.
- 대응 전략은 단정형 지시가 아니라 미팅 준비용 제안으로 보이게 한다.

### 8.5 소프트웨어별 가치 맵

- 소프트웨어명 기준으로 기능 목록을 보여준다.
- 기능명, 기능 요약, 가치 키워드, 관련 니즈 키워드를 표시한다.
- `related_need_keywords`는 직접 연결이 아니라 느슨한 키워드 연결임을 화면에 작게 표시한다.
- 특정 소프트웨어명을 코드나 화면 구조에 하드코딩하지 않는다.

### 8.6 마케팅 활용

별도 화면으로 구성한다.

표시 후보:

- 반복적으로 등장하는 고객 니즈 키워드
- 산업군 또는 고객군별 관심 주제
- 소프트웨어별 가치 메시지 후보
- 세미나 또는 기술 콘텐츠 주제로 활용 가능한 니즈
- 제안서나 고객 맞춤 기능 소개에 활용 가능한 기능/가치 연결

주의:

- 마케팅 활용 후보는 확정 캠페인이나 확정 고객 수요처럼 표현하지 않는다.
- 고객 내부 의사결정, 예산, 구매 가능성을 추정하지 않는다.
- 낮은 confidence 항목은 활용 전 담당자 확인이 필요하다고 표시한다.

## 9. 로그인/접근 안내 화면 기준

로그인 또는 접근 안내 화면 구성:

- 라온엑스 로고
- 서비스명: `고객 니즈-기능 매칭 대시보드`
- 계정 입력
- 비밀번호 입력
- 로그인 버튼
- 오류 메시지
- 내부 시스템 안내 문구

권한 안내:

- `admin`: 조회, 데이터 수정, 매칭 신뢰도/메모 등 운영 정보 관리, 운영 반영 가능
- `viewer`: 조회 전용

주의:

- 대시보드는 사내망 내부 전용이다.
- 외부 인터넷 접속은 범위에서 제외한다.
- 인증 정보를 코드나 Git에 노출하지 않는다.

## 10. PDF/인쇄 출력 기준

PDF/인쇄 출력은 1차 개발 필수는 아니지만, 향후 마케팅 자료 기획 또는 고객 미팅 준비자료로 출력될 수 있습니다.

PDF 구성 후보:

1. 표지/제목
2. 고객사 또는 고객군 요약
3. 주요 니즈
4. 추천 소프트웨어/기능
5. 가치 메시지
6. 마케팅 활용 후보 또는 다음 미팅 추천 질문
7. confidence 낮음, 데이터 누락, 확인 필요 사항
8. source file 및 report date

## 11. 금지 사항

- 라온엑스 로고 색상을 임의로 변경하지 않는다.
- 데이터 근거 없는 고객 의도, 구매 가능성, 예산, 도입 의지를 추정하지 않는다.
- `confidence`가 낮은 매칭을 확정 추천처럼 표시하지 않는다.
- 마케팅 활용 후보를 확정 캠페인 또는 확정 고객 수요처럼 표현하지 않는다.
- `dashboard_visible=FALSE` 항목을 기본 화면에 노출하지 않는다.
- `review_status` UI를 1차 범위에서 만들지 않는다.
- `sensitivity_level`을 표시/필터/배지 기준으로 사용하지 않는다.
- 고객사명과 담당자 정보를 불필요하게 화면에 과도하게 노출하지 않는다.
- 실제 고객 정보를 예제 화면이나 테스트 이미지에 임의로 사용하지 않는다.
- admin 수정 UI를 viewer에게 노출하지 않는다.
- 인증 정보를 코드 예시 또는 화면 예시에 노출하지 않는다.
- 내부 시스템을 외부 공개용 홍보 페이지처럼 보이게 만들지 않는다.
