# DESIGN.md — 라온엑스 고객 니즈-기능 매칭 대시보드 디자인 기준

> 목적: Codex가 `knowledge_for_marketing_dashboard`의 Streamlit 대시보드, 조회 화면, 검토 상태 표시, 권한 안내, 향후 PDF/인쇄용 출력물을 만들 때 참고할 디자인 기준입니다. 이 문서는 단순히 예쁜 화면을 만들기 위한 문서가 아니라, 마케팅 자료 기획과 고객 미팅 준비에 필요한 고객 니즈, 추천 소프트웨어 기능, 제안 메시지, 다음 확인 질문을 빠르게 이해하고 바로 활용할 수 있도록 만드는 기준 문서입니다.

---

## 1. 디자인 목표

이 대시보드는 라온엑스 내부에서 고객 방문보고서와 소프트웨어 기능 정보를 연결해 활용하기 위한 사내 지식 검색형 대시보드입니다.

디자인 목표는 크게 두 가지입니다.

1. **마케팅 활용**
   - 고객 니즈와 소프트웨어 기능 정보를 바탕으로 세미나, 제안서, 고객 맞춤 기능 소개, 기술 콘텐츠 기획에 활용할 수 있게 한다.
   - 반복적으로 등장하는 니즈, 소프트웨어별 가치 포인트, 고객군별 관심 주제를 빠르게 파악할 수 있게 한다.
   - 단순 기능 나열이 아니라 “어떤 고객 문제에 어떤 가치 메시지를 연결할 수 있는가”가 보이게 한다.

2. **고객 미팅 준비**
   - 고객 미팅 전 3~5분 안에 핵심 정보를 파악한다.
   - 고객 니즈 → 추천 기능 → 제안 가치 → 다음 미팅 질문 흐름이 한눈에 보이게 한다.
   - 검토 필요, 낮은 신뢰도, 비공개 항목을 숨기지 않고 명확히 표시한다.

공통 목표:

- 영업 담당자와 엔지니어가 모두 이해할 수 있는 업무형 문구를 사용한다.
- 내부 업무 시스템처럼 신뢰감 있고 절제된 화면을 만든다.
- 마케팅 활용과 고객 미팅 준비 모두에서 근거 없는 과장 표현을 피한다.

대시보드의 인상:

- 현대적인 SaaS 대시보드
- 마케팅 기획과 고객 미팅 준비를 함께 지원하는 업무 화면
- 고객 니즈와 소프트웨어 기능을 근거 기반으로 연결하는 화면
- 고객사별 대응 전략과 추천 질문을 빠르게 확인하는 화면
- 화려한 홍보 페이지가 아니라 신뢰감 있는 내부 시스템

## 2. 설계 원칙

| 원칙 | 설명 |
|---|---|
| 마케팅 활용 지원 | 반복 니즈, 가치 메시지, 소프트웨어별 강점, 고객군별 관심 주제를 콘텐츠·제안 기획에 활용할 수 있어야 한다. |
| 미팅 준비 지원 | 모든 화면은 “고객 미팅 전에 무엇을 확인해야 하는가?”에 답해야 한다. |
| 근거 기반 | 고객 니즈, 추천 기능, 제안 메시지는 방문보고서, wiki, export CSV에 근거해야 한다. |
| 검토 상태 노출 | `검토 필요`, `신뢰도 낮음`, `데이터 누락`은 숨기지 않는다. |
| 민감도 경고 | `sensitivity_level=비공개`는 숨김 기준이 아니라 경고 배지 기준으로 표시한다. |
| 노출 기준 준수 | `dashboard_visible=FALSE` 또는 이에 준하는 값은 기본 화면에서 숨긴다. |
| 단정 금지 | 구매 가능성, 예산, 도입 의지, 고객 내부 의사결정 상황을 추정하지 않는다. |
| 검색 중심 | 1차 개발은 복잡한 필터보다 검색창과 핵심 요약 중심으로 구성한다. |
| 확장 가능 | 소프트웨어명은 하드코딩하지 않고 CSV의 `software` 값을 기준으로 처리한다. |

## 3. 브랜드와 로고 기준

라온엑스 CI는 그레이 계열의 안정감 있는 로고와 청록색 포인트 `X`를 사용합니다. 대시보드도 이 방향을 따릅니다.

로고 사용 원칙:

- 라온엑스 로고는 실제 제공된 PNG 파일을 우선 사용한다.
- 로고 파일이 없으면 외부 URL에서 임의로 가져오지 않고 텍스트 로고 또는 임시 placeholder로 처리한다.
- 로고 원본 색상을 변경하지 않는다.
- 로고를 찌그러뜨리거나 과도하게 확대하지 않는다.
- 모든 카드마다 로고를 반복하지 않는다.

권장 로고 파일 경로:

```text
assets/logo/raonx_logo_light_bg.png
assets/logo/raonx_logo_dark_bg.png
assets/logo/raonx_logo_mono.png
```

사용 위치:

- 대시보드 상단 헤더 좌측
- 로그인 또는 접근 안내 화면 상단
- PDF/인쇄 출력물 상단 좌측
- 오류 또는 빈 상태 화면의 작은 브랜드 요소

로고 여백:

- 로고 주변에는 로고 높이의 최소 0.5배 이상 여백을 둔다.
- 헤더 로고 높이는 28–40px 범위를 우선한다.
- 로그인/접근 안내 화면에서는 120–180px 폭으로 사용한다.

## 4. 색상 토큰

### 4.1 기본 색

| 토큰 | HEX | 용도 |
|---|---|---|
| `canvas` | `#F6F8FA` | 기본 배경 |
| `surface` | `#FFFFFF` | 카드, 표, 패널 |
| `surface-soft` | `#F1F4F5` | 필터 영역, 보조 패널 |
| `ink` | `#1F2426` | 제목, 핵심 텍스트 |
| `body` | `#3F4749` | 본문 텍스트 |
| `muted` | `#6E7779` | 보조 설명, 날짜, 출처 |
| `hairline` | `#DDE3E5` | 카드 경계, 표 구분선 |
| `disabled` | `#B8C0C2` | 비활성 텍스트, 비활성 버튼 |

### 4.2 라온엑스 포인트 색

| 토큰 | HEX | 용도 |
|---|---|---|
| `raonx-gray-dark` | `#5B6261` | 제목, 보조 강조, 네비게이션 |
| `raonx-gray-light` | `#B8BEC0` | 보조선, 배경 강조, 비활성 상태 |
| `raonx-cyan` | `#36BFC7` | 주요 강조, 선택 상태, 핵심 액션 |
| `raonx-cyan-soft` | `#E6F8FA` | 선택된 필터, 정보 배지, 추천 기능 강조 |
| `navy-ink` | `#263238` | 안정감 있는 헤더/텍스트 |

주의:

- 청록색은 주요 포인트와 선택 상태에만 사용한다.
- 청록색을 모든 카드와 버튼에 남발하지 않는다.
- 실제 라온엑스 로고 PNG에서 확인되는 색상을 우선한다.

### 4.3 상태 색

| 상태 | 색상 | 용도 |
|---|---|---|
| 검토 완료 | `#20A67A` | 검토 완료, confidence 높음 |
| 검토 필요 | `#D97706` | 검토 필요, 확인 필요 |
| 위험/비공개 | `#DC2626` | 비공개, confidence 낮음, 오류 |
| 정보/참고 | `#64748B` | 참고, 보조 정보 |
| 데이터 없음 | `#CBD5D9` | 결측, 데이터 없음 |

빨간색은 다음 상황에만 사용한다.

- `sensitivity_level=비공개`
- 로그인 실패 또는 접근 실패
- CSV 로딩 실패
- 필수 컬럼 누락
- 데이터 관계 오류
- `confidence=낮음`
- 운영상 즉시 확인이 필요한 상태

## 5. 타이포그래피

기본 폰트:

```text
Pretendard, Apple SD Gothic Neo, Noto Sans KR, sans-serif
```

숫자/ID 후보:

```text
Inter, Pretendard, sans-serif
```

권장 크기:

| 요소 | 기준 |
|---|---|
| 페이지 제목 | 28–34px, 700 |
| 섹션 제목 | 20–24px, 700 |
| 카드 제목 | 14–15px, 600 |
| 본문 | 14–16px, 400–500, line-height 1.55 |
| 캡션/메타 | 12–13px, 400, muted |
| KPI 숫자 | 32–44px, 700 |
| 표 텍스트 | 13–14px, 400–500 |
| ID 값 | 12–13px, 500 |
| 버튼 | 14px, 600 |

문장 원칙:

- 짧고 명확하게 쓴다.
- 기능 설명보다 업무 판단에 도움이 되는 표현을 우선한다.
- 고객이 이미 답한 것처럼 표현하지 않는다.
- 데이터에 없는 고객 의도나 구매 가능성을 쓰지 않는다.

좋은 문장:

- “검토 필요 매칭이 있습니다.”
- “이 고객 니즈와 연결된 추천 기능을 확인하세요.”
- “비공개 항목은 내부 공유 범위를 확인한 뒤 사용하세요.”
- “다음 미팅에서 확인할 질문을 검토하세요.”
- “confidence가 낮은 매칭은 담당자 확인 후 제안에 활용하세요.”
- “반복적으로 등장하는 니즈를 마케팅 콘텐츠 주제로 검토하세요.”

피해야 할 문장:

- “이 고객은 구매 가능성이 높습니다.”
- “이 기능을 제안하면 수주될 것 같습니다.”
- “고객이 반드시 이 기능을 원합니다.”
- “예산이 충분할 것으로 보입니다.”
- “검토 상태가 없지만 바로 제안해도 됩니다.”

## 6. 레이아웃 원칙

대시보드는 개인 PC 브라우저와 노트북 화면을 우선합니다.

기준:

- 기본 화면 폭은 1280px 이상 데스크톱 브라우저를 기준으로 한다.
- 12컬럼 그리드 또는 카드 기반 반응형 레이아웃을 사용한다.
- 모바일은 필수 우선순위가 아니지만 깨지지 않게 유지한다.
- 첫 화면은 `Home`을 가장 우선한다.
- 첫 화면 상단에는 라온엑스 로고, 서비스명, 마지막 데이터 업데이트 시간을 표시한다.
- 검색창은 사용자가 가장 먼저 보게 상단에 배치한다.
- 필터는 처음부터 과도하게 만들지 않고, 1차 개발에서는 일반 검색창 중심으로 구성한다.
- 카드 간격은 16–24px, 섹션 간격은 32–48px을 기본으로 한다.
- 고객사명과 담당자는 표시하되, 불필요하게 큰 시각 강조를 하지 않는다.
- `sensitivity_level=비공개`는 경고 배지로 분명히 구분한다.

Home 권장 순서:

1. 라온엑스 로고 + 대시보드 제목 + 마지막 업데이트 시간
2. 전역 검색창
3. 핵심 요약 카드
4. 최근 고객 니즈
5. 자주 등장한 니즈 키워드
6. 검토 필요 매칭 현황
7. 고객사별 대응 전략 바로가기
8. 소프트웨어별 가치 포인트 바로가기
9. 데이터 품질 경고
10. 비공개 데이터 표시 안내

## 7. 핵심 컴포넌트 기준

### 7.1 카드

카드 기본:

- 배경: `surface`
- 테두리: 1px `hairline`
- 라운드: 16px
- 그림자: `0 8px 24px rgba(15, 23, 42, 0.06)` 이하
- 내부 여백: 20–24px
- 카드마다 가능한 경우 한 줄 요약 포함

고객 니즈 카드 구조:

1. 고객사명
2. 산업군
3. 니즈 키워드
4. 니즈 요약
5. 보고서 날짜
6. 담당자
7. 민감도 배지
8. 연결된 추천 기능 개수

기능 매칭 카드 구조:

1. 니즈 키워드
2. 소프트웨어명
3. 기능명
4. 가치 메시지
5. confidence 배지
6. review_status 배지
7. last_reviewed
8. 다음 확인 질문 또는 참고 안내

고객사별 대응 전략 카드 구조:

1. 고객사명
2. 주요 니즈
3. 추천 소프트웨어
4. 접근 방향
5. 제안 메시지
6. 다음 미팅 추천 질문
7. review_status 배지
8. next_action

마케팅 활용 카드 구조:

1. 반복 니즈 키워드
2. 관련 산업군 또는 고객군
3. 연결 가능한 소프트웨어/기능
4. 가치 메시지 후보
5. 콘텐츠 또는 세미나 주제 후보
6. 근거 데이터 또는 출처
7. 검토 필요 여부

### 7.2 버튼

- 기본 CTA: `raonx-cyan` 배경 + 흰색 텍스트
- 보조 버튼: 흰색 배경 + `raonx-gray-dark` 텍스트 + 1px `hairline`
- 위험 버튼: 빨간색 계열, 로그아웃·초기화·오류 작업에만 사용
- 라운드: 10–12px
- 버튼 문구는 짧고 동작 중심으로 쓴다.

예:

- “조회”
- “검색”
- “필터 초기화”
- “상세 보기”
- “데이터 새로고침”
- “로그인”

### 7.3 배지

| 데이터 | 표시 |
|---|---|
| `review_status=검토 완료` | 초록 계열 배지 |
| `review_status=검토 필요` | 주황 계열 배지 |
| `confidence=높음` | 초록 계열 배지 |
| `confidence=중간` | 주황 계열 배지 |
| `confidence=낮음` | 빨강 계열 배지 |
| `sensitivity_level=공개 가능` | 초록 또는 정보 계열 배지 |
| `sensitivity_level=비공개` | 빨강 계열 배지 |
| 데이터 없음 | 회색 배지 |

배지 문구 예:

```text
검토 완료
검토 필요
높음
중간
낮음
공개 가능
비공개
데이터 없음
```

### 7.4 필터

1차 개발 기본 필터 후보:

- 전역 검색창
- 고객사명
- 산업군
- 니즈 키워드
- 소프트웨어
- review_status
- confidence
- sensitivity_level

원칙:

- 초기에는 드롭다운을 과도하게 늘리지 않는다.
- 검색창 중심으로 탐색한다.
- 1차 개발 후 사용자 피드백을 받아 필터를 확장한다.
- 필터 선택 상태는 `raonx-cyan-soft` 배경 또는 `raonx-cyan` 테두리로 표시한다.

### 7.5 테이블

- 고객사명, 니즈 키워드, 소프트웨어명은 좌측 정렬한다.
- ID, 날짜, 상태값은 중앙 또는 좌측 정렬한다.
- 긴 텍스트는 기본적으로 2~3줄까지만 보이고 상세 보기에서 전체 표시한다.
- 헤더는 가능하면 고정한다.
- 정렬 가능하게 만든다.
- `review_status`, `confidence`, `sensitivity_level`은 텍스트만 쓰지 말고 배지로 표시한다.
- 너무 많은 컬럼을 한 번에 보여주지 않는다.
- CSV 원본 컬럼명을 그대로 노출하기보다 사용자가 이해하기 쉬운 한글 라벨을 사용한다.

## 8. 차트 기준

차트는 많을 필요가 없습니다. 이 대시보드는 매출 대시보드가 아니라 고객 니즈와 기능 매칭을 빠르게 찾기 위한 지식 검색형 대시보드입니다.

권장 차트:

| 목적 | 권장 표시 |
|---|---|
| 최근 고객 니즈 현황 | 날짜별 리스트 카드 또는 고객사별 리스트 |
| 자주 등장한 니즈 | 키워드 빈도 막대 차트 |
| 검토 필요 매칭 | 상태별 카드 또는 가로 막대 |
| 소프트웨어별 기능 수 | 가로 막대 차트 |
| confidence 분포 | 배지 요약 카드 또는 간단한 분포 차트 |
| 고객사별 대응 전략 수 | 고객사별 리스트 또는 막대 차트 |
| 마케팅 콘텐츠 후보 | 니즈 키워드별 카드 또는 리스트 |

차트 원칙:

- 한 차트에 너무 많은 색을 쓰지 않는다.
- 핵심 강조색 `raonx-cyan`은 선택 상태 또는 주요 지표에 우선 사용한다.
- 검토 필요와 비공개는 주황/빨강 계열로 일관되게 표시한다.
- 범례는 차트 아래 또는 오른쪽에 작게 배치한다.
- 차트 제목은 단순형, 부제는 해석형으로 작성한다.
- 차트가 판단을 방해하면 표나 카드가 우선이다.

예:

```text
제목: 검토 필요 매칭
부제: confidence가 낮거나 검토 필요 상태인 항목을 먼저 확인하세요.
```

## 9. 화면별 디자인 기준

### 9.1 Home

Home은 1차 개발의 최우선 화면입니다.

첫 화면에서 보여줄 것:

1. 라온엑스 로고와 서비스명
2. 마지막 데이터 업데이트 시간
3. 전체 고객 니즈 수
4. 전체 매칭 수
5. 검토 필요 매칭 수
6. confidence 낮음 항목 수
7. 비공개 항목 수
8. 최근 고객 니즈
9. 자주 등장한 니즈 키워드
10. 마케팅 활용 후보 니즈 또는 가치 메시지
11. 데이터 품질 경고
12. 고객사별 대응 전략 바로가기

Home의 역할:

- 전체 현황을 빠르게 파악한다.
- 마케팅에 활용할 반복 니즈와 가치 메시지 후보를 찾는다.
- 고객 미팅 전 검토가 필요한 항목을 먼저 찾는다.
- 다음 화면으로 이동하는 출발점이 된다.

### 9.2 고객 니즈 검색

- 전역 검색창을 가장 상단에 둔다.
- 고객사명, 산업군, 니즈 키워드, 니즈 요약을 함께 검색할 수 있게 한다.
- 결과는 카드 또는 표로 제공한다.
- 고객사명과 담당자는 표시하되 과도하게 강조하지 않는다.
- `sensitivity_level`과 `dashboard_visible` 상태를 명확히 표시한다.
- `dashboard_visible=FALSE` 또는 이에 준하는 항목은 기본 결과에서 숨긴다.

### 9.3 니즈-기능-가치 매칭

- `need_id`와 `feature_id` 연결 결과를 보여준다.
- 고객 니즈 → 추천 소프트웨어 → 기능 → 가치 메시지 순서로 읽히게 한다.
- `review_status`와 `confidence`를 반드시 배지로 표시한다.
- 검토 필요 항목은 상단 또는 별도 섹션에서 쉽게 찾게 한다.
- 검토 완료가 아닌 매칭은 확정 추천처럼 보이지 않게 한다.
- 가치 메시지 아래에 “다음 확인 질문” 또는 “활용 전 확인 사항”을 함께 표시한다.

### 9.4 고객사별 대응 전략

- 고객사명 기준으로 검색한다.
- `main_needs`, `recommended_software`, `approach_details`, `suggested_questions`, `proposal_message`, `next_action`을 순서대로 보여준다.
- `review_status`는 검토 필요/검토 완료로 명확히 표시한다.
- 영업 미팅 전 바로 읽을 수 있는 구조로 만든다.
- 대응 전략은 단정형 지시가 아니라 미팅 준비용 제안으로 보이게 한다.

### 9.5 소프트웨어별 가치 맵

- 소프트웨어명 기준으로 기능 목록을 보여준다.
- 기능명, 기능 요약, 가치 키워드, 관련 니즈 키워드를 표시한다.
- `related_need_keywords`는 직접 연결이 아니라 느슨한 키워드 연결임을 화면에 작게 표시한다.
- 소프트웨어가 추가될 수 있게 확장 가능한 레이아웃으로 만든다.
- 특정 소프트웨어명을 코드나 화면 구조에 하드코딩하지 않는다.

### 9.6 마케팅 활용 관점

별도 화면으로 만들지 않더라도 Home, 니즈-기능-가치 매칭, 소프트웨어별 가치 맵에서 마케팅 활용 관점을 확인할 수 있어야 합니다.

표시 후보:

- 반복적으로 등장하는 고객 니즈 키워드
- 산업군 또는 고객군별 관심 주제
- 소프트웨어별 가치 메시지 후보
- 세미나 또는 기술 콘텐츠 주제로 활용 가능한 니즈
- 제안서나 고객 맞춤 기능 소개에 활용 가능한 기능/가치 연결

주의:

- 마케팅 활용 후보는 확정 캠페인이나 확정 고객 수요처럼 표현하지 않는다.
- 고객 내부 의사결정, 예산, 구매 가능성을 추정하지 않는다.
- 검토 필요 상태의 매칭은 마케팅 활용 전 담당자 검토가 필요하다고 표시한다.

## 10. 로그인/접근 안내 화면 기준

운영 공유 전 로그인과 권한 구분은 적용 대상입니다. 단, 1차 개발에서 로그인 방식과 admin 수정값 저장 위치가 확정되지 않았으면 임의 구현하지 않습니다.

로그인 또는 접근 안내 화면 구성 후보:

- 라온엑스 로고
- 서비스명: `고객 니즈-기능 매칭 대시보드`
- 아이디 입력
- 비밀번호 입력
- 로그인 버튼
- 오류 메시지
- 내부 시스템 안내 문구

스타일:

- 밝은 배경
- 중앙 정렬 카드
- 과한 애니메이션 금지
- 라온엑스 청록색을 로그인 버튼에 사용
- 로그인 실패는 빨간색으로 명확히 표시하되, 상세 보안 정보는 노출하지 않음
- 밝은 배경에서는 밝은 바탕용 라온엑스 로고 PNG 사용

안내 문구 예:

```text
라온엑스 내부 사용자 전용 시스템입니다.
```

권한 안내:

- `admin`: 데이터 수정, 검토 상태/신뢰도 관리, 운영 반영 가능
- `viewer`: 조회 전용
- 1차 개발에서는 팀원은 기본적으로 `viewer` 권한으로 본다.

## 11. PDF/인쇄 출력 기준

PDF/인쇄 출력은 1차 개발 필수는 아니지만, 향후 마케팅 자료 기획 또는 고객 미팅 준비자료로 출력될 수 있습니다.

원칙:

- A4 가로 방향을 우선 고려한다.
- 배경색은 너무 진하지 않게 한다.
- 카드 그림자는 인쇄 시 최소화한다.
- 표와 상세 텍스트가 잘리지 않게 페이지 분리를 고려한다.
- 로고, 기준 데이터 업데이트일, 출력일을 상단에 표시한다.
- PDF에서는 인터랙션 요소를 제거하고 정적인 설명을 추가한다.
- 비공개, 검토 필요, 낮은 confidence는 흑백 출력에서도 구분 가능하게 아이콘 또는 라벨을 함께 사용한다.
- 내부 검토용 마케팅 자료 또는 고객 미팅 준비자료 기준으로 작성한다.

PDF 구성 후보:

1. 표지/제목
2. 고객사 또는 고객군 요약
3. 주요 니즈
4. 추천 소프트웨어/기능
5. 가치 메시지
6. 마케팅 활용 후보 또는 다음 미팅 추천 질문
7. 검토 필요 및 주의사항
8. source file 및 report date

## 12. 대시보드 출력 기준

대시보드는 예쁘기보다 마케팅 기획과 고객 미팅 준비가 빨라야 합니다.

첫 화면에서 보여줄 것:

1. 전체 고객 니즈 수
2. 전체 니즈-기능 매칭 수
3. 검토 필요 매칭 수
4. 검토 완료 매칭 수
5. confidence 낮음 항목 수
6. 비공개 항목 수
7. 최근 고객 니즈
8. 자주 등장한 니즈
9. 소프트웨어별 기능 수
10. 마케팅 활용 후보 니즈 또는 가치 메시지
11. 데이터 누락/오류/관계 불일치 경고

모든 결론에는 가능한 한 다음 두 가지가 같이 있어야 합니다.

```text
근거 데이터 + 다음 확인 항목
```

카드 한 줄 요약 예:

- `검토 필요 매칭이 있습니다. 담당자가 확인 후 상태를 갱신하세요.`
- `비공개 항목은 고객 미팅 자료로 활용하기 전에 공유 범위를 확인하세요.`
- `confidence가 낮은 매칭은 제안 메시지로 바로 사용하지 마세요.`
- `반복적으로 등장하는 니즈를 마케팅 콘텐츠 주제로 검토하세요.`
- `이 고객 니즈와 연결된 추천 기능을 확인하세요.`

## 13. 금지 사항

- 라온엑스 로고 색상을 임의로 변경하지 않는다.
- 로고를 찌그러뜨리거나 과도하게 확대하지 않는다.
- 청록색을 모든 요소에 남발하지 않는다.
- 빨간색을 과다 사용하지 않는다.
- 데이터 근거 없는 고객 의도, 구매 가능성, 예산, 도입 의지를 추정하지 않는다.
- `confidence`가 낮은 매칭을 확정 추천처럼 표시하지 않는다.
- `review_status=검토 필요`인 항목을 검토 완료처럼 보이게 하지 않는다.
- `sensitivity_level=비공개` 항목을 일반 공개 가능 항목처럼 보이게 하지 않는다.
- `dashboard_visible=FALSE` 항목을 기본 화면에 노출하지 않는다.
- 고객사명과 담당자 정보를 불필요하게 화면에 과도하게 노출하지 않는다.
- 실제 고객 정보를 예제 화면이나 테스트 이미지에 임의로 사용하지 않는다.
- 마케팅 활용 후보를 확정 캠페인 또는 확정 고객 수요처럼 표현하지 않는다.
- 차트를 많이 넣는 대신 핵심 검색과 판단이 안 보이게 만들지 않는다.
- 과한 애니메이션, 과한 그라데이션, 장식용 그래픽을 사용하지 않는다.
- 내부 시스템을 외부 공개용 홍보 페이지처럼 보이게 만들지 않는다.
- 로그인 방식, admin 수정 저장 방식이 확정되지 않았는데 확정된 기능처럼 표현하지 않는다.
