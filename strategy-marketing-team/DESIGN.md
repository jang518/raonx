# DESIGN.md — 라온엑스 고객 니즈-기능 매칭 대시보드 디자인 기준

> 목적: Codex가 `knowledge_for_marketing` 대시보드, 로그인 화면, 데이터 조회 화면, 검토 상태 표시, 향후 PDF/인쇄용 출력물을 만들 때 참고할 라온엑스 디자인 기준입니다. 이 문서는 예쁜 화면을 만들기 위한 문서가 아니라, 영업·엔지니어가 고객 미팅 전에 고객 니즈와 추천 소프트웨어 기능을 빠르게 이해하고 바로 활용할 수 있도록 만드는 기준 문서입니다.

## 1. 브랜드/서비스 인상

이 대시보드는 라온엑스 내부에서 고객 방문보고서와 소프트웨어 기능 정보를 연결해 활용하기 위한 사내 지식 검색형 대시보드다.

핵심 이미지는 다음과 같다.

  * 현대적인 SaaS 대시보드
  * 영업·엔지니어가 미팅 전 빠르게 확인할 수 있는 준비 화면
  * 고객 니즈와 소프트웨어 기능을 근거 기반으로 연결하는 업무용 화면
  * 고객사별 대응 전략과 추천 질문을 한눈에 보여주는 보고서형 화면
  * 과하게 화려하지 않고 신뢰감 있는 내부 업무 시스템

라온엑스 CI는 그레이 계열의 안정감 있는 로고와 청록색 포인트 `X`를 사용한다. 대시보드도 이 방향을 따른다. 전체 화면은 밝고 깔끔하게 구성하되, 핵심 매칭 정보, 검토 필요 상태, 비공개 민감도 표시는 명확하게 보이게 한다.

라온엑스 로고는 반드시 실제 제공된 PNG 파일을 우선 사용한다.

권장 로고 사용:

  * 밝은 배경: `assets/logo/raonx_logo_light_bg.png`
  * 진한 배경 또는 로그인 화면의 어두운 영역: `assets/logo/raonx_logo_dark_bg.png`
  * 단색 또는 보조 워터마크: `assets/logo/raonx_logo_mono.png`

로고 파일은 원본 PNG를 `assets/logo/`에 복사해 사용한다. 코드에서 외부 URL로 로고를 불러오지 않는다.

## 2. 디자인 키워드

  * 현대적: SaaS 대시보드처럼 깔끔하고 정돈된 인터페이스
  * 신뢰감: 고객 니즈, 추천 기능, 검토 상태, 근거 파일을 명확하게 표시
  * 빠른 검색: 고객사명, 산업군, 니즈 키워드, 소프트웨어명을 빠르게 찾을 수 있게 구성
  * 회의 준비 친화적: 고객 미팅 전 3~5분 안에 핵심 정보를 파악할 수 있게 구성
  * 절제된 강조: 중요한 매칭과 검토 필요 항목은 강조하지만 색상과 장식은 과하지 않게 사용
  * 데이터 중심: 감성 문구보다 고객 니즈, 기능 가치, 제안 메시지, 다음 질문 중심

## 3. 색상 토큰

### 기본 색

  * `canvas` — #F6F8FA: 기본 배경. 밝고 중립적인 업무용 배경.
  * `surface` — #FFFFFF: 카드, 표, 대시보드 패널.
  * `surface-soft` — #F1F4F5: 보조 패널, 필터 영역, 로그인 박스 배경.
  * `ink` — #1F2426: 제목과 핵심 텍스트.
  * `body` — #3F4749: 본문 텍스트.
  * `muted` — #6E7779: 보조 설명, 캡션, 날짜, source file.
  * `hairline` — #DDE3E5: 카드 경계, 표 구분선.
  * `disabled` — #B8C0C2: 비활성 텍스트, 비활성 버튼.

### 브랜드 포인트

라온엑스 CI 기준 색상:

  * `raonx-gray-dark` — #5B6261: 라온엑스 로고의 진한 그레이 계열. 제목, 보조 강조, 네비게이션에 사용.
  * `raonx-gray-light` — #B8BEC0: 라온엑스 로고의 밝은 그레이 계열. 보조선, 배경 강조, 비활성 상태에 사용.
  * `raonx-cyan` — #36BFC7: 라온엑스 CI의 청록 포인트. 주요 강조, 선택 상태, 핵심 액션에 사용.
  * `raonx-cyan-soft` — #E6F8FA: 청록색의 약한 배경. 선택된 필터, 정보 배지, 추천 기능 강조에 사용.
  * `navy-ink` — #263238: 전문성과 안정감을 주는 진한 텍스트/헤더 색상.

CI PDF에 표시된 CMYK 기준도 함께 참고한다.

  * 진한 그레이: `C 10 M 00 Y 10 K 75`
  * 청록 포인트: `C 66 M 00 Y 25 K 00` 계열
  * 밝은 그레이: `C 04 M 00 Y 04 K 30`

실제 화면 구현에서는 위 CMYK를 직접 쓰기보다 PNG 로고에서 확인되는 웹용 HEX 토큰을 우선 사용한다.

### 데이터 시각화 색

  * 고객 니즈: `raonx-cyan` #36BFC7
  * 추천 기능: #4F8F9A
  * 니즈-기능 매칭: #2F6F7E
  * 고객 대응 전략: #5661A9
  * 다음 미팅 질문: #64748B
  * 검토 완료: #20A67A
  * 검토 필요: #D97706
  * confidence 높음: #20A67A
  * confidence 중간: #D97706
  * confidence 낮음: #DC2626
  * 비공개: #DC2626
  * 공개 가능: #20A67A
  * 데이터 없음: #CBD5D9
  * 정보/참고: #64748B

### 경고 색상 원칙

경고와 위험 신호는 빨간색으로 명확히 표시한다. 단, 빨간색을 과다 사용하지 않는다.

  * 치명적 오류: #DC2626
  * 주의 필요: #D97706
  * 참고/확인 필요: #64748B
  * 정상/양호: #20A67A

빨간색은 다음 상황에만 사용한다.

  * `sensitivity_level = 비공개`
  * 로그인 실패
  * CSV 로딩 실패
  * 필수 컬럼 누락
  * 데이터 관계 오류
  * confidence 낮음
  * 운영상 즉시 확인이 필요한 상태

## 4. 타이포그래피

  * 기본 폰트: Pretendard, Apple SD Gothic Neo, Noto Sans KR, sans-serif
  * 숫자/ID 폰트: Inter, Pretendard, sans-serif
  * 제목: 28–34px, 700, 자간 -0.02em
  * 섹션 제목: 20–24px, 700
  * 카드 제목: 14–15px, 600
  * 본문: 14–16px, 400–500, line-height 1.55
  * 캡션/메타: 12–13px, 400, `muted`
  * KPI 숫자: 32–44px, 700, line-height 1.1
  * 표 텍스트: 13–14px, 400–500
  * ID 값: 12–13px, 500, monospace 후보
  * 버튼: 14px, 600

문장은 짧고 명확하게 쓴다. 영업·엔지니어가 고객 미팅 전에 바로 사용할 수 있는 문장을 우선한다.

좋은 문장:

  * “검토 필요 매칭이 있습니다.”
  * “이 고객 니즈와 연결된 추천 기능을 확인하세요.”
  * “비공개 항목은 내부 공유 범위를 확인한 뒤 사용하세요.”
  * “다음 미팅에서 확인할 질문을 검토하세요.”

피해야 할 문장:

  * “이 고객은 구매 가능성이 높습니다.”
  * “이 기능을 제안하면 수주될 것 같습니다.”
  * “고객이 반드시 이 기능을 원합니다.”
  * “상황이 좋아 보입니다.”

## 5. 레이아웃 원칙

대시보드는 개인 PC 브라우저와 노트북 화면을 우선한다.

  * 기본 화면 폭은 1280px 이상 데스크톱 브라우저를 기준으로 한다.
  * 12컬럼 그리드 또는 카드 기반 반응형 레이아웃을 사용한다.
  * 첫 화면은 `Home`을 가장 우선한다.
  * 첫 화면 상단에는 라온엑스 로고, 서비스명, 마지막 데이터 업데이트 시간을 표시한다.
  * 검색창은 사용자가 가장 먼저 보게 상단에 배치한다.
  * 필터는 처음부터 과도하게 만들지 않고, 1차 개발에서는 일반적인 검색창 중심으로 구성한다.
  * 카드 간격은 16–24px, 섹션 간격은 32–48px을 기본으로 한다.
  * 모바일은 필수 우선순위가 아니지만, 깨지지 않는 반응형은 유지한다.
  * 고객사명과 담당자는 표시하되, 불필요하게 큰 시각 강조를 하지 않는다.
  * `sensitivity_level = 비공개`는 화면에 표시하되 경고 배지로 분명히 구분한다.

첫 화면 권장 순서:

  1. 라온엑스 로고 + 대시보드 제목 + 마지막 업데이트 시간
  2. 전역 검색창
  3. 핵심 요약 카드
  4. 최근 고객 니즈
  5. 자주 등장한 니즈 키워드
  6. 검토 필요 매칭 현황
  7. 고객사별 대응 전략 바로가기
  8. 데이터 품질 경고
  9. 비공개 데이터 표시 안내

## 6. 컴포넌트 기준

### 카드

  * 배경: `surface`
  * 테두리: 1px `hairline`
  * 라운드: 16px
  * 그림자: 약하게 사용
    * `0 8px 24px rgba(15, 23, 42, 0.06)` 이하
  * 카드 내부 여백: 20–24px
  * 카드 제목과 핵심 정보는 명확히 분리
  * 카드마다 가능한 경우 한 줄 요약을 포함

고객 니즈 카드 구조:

  1. 고객사명
  2. 산업군
  3. 니즈 키워드
  4. 니즈 요약
  5. 보고서 날짜
  6. 담당자
  7. 민감도 배지
  8. 연결된 추천 기능 개수

기능 매칭 카드 구조:

  1. 니즈 키워드
  2. 소프트웨어명
  3. 기능명
  4. 가치 메시지
  5. confidence 배지
  6. review_status 배지
  7. last_reviewed

### 버튼

  * 기본 CTA: `raonx-cyan` 배경 + 흰색 텍스트
  * 보조 버튼: 흰색 배경 + `raonx-gray-dark` 텍스트 + 1px `hairline`
  * 위험 버튼: 빨간색 계열. 로그아웃, 초기화, 오류 작업에만 사용
  * 라운드: 10–12px
  * 버튼 문구는 짧고 동작 중심으로 쓴다.

예:

  * “조회”
  * “검색”
  * “필터 초기화”
  * “상세 보기”
  * “코멘트 남기기”
  * “데이터 새로고침”
  * “로그인”

### 배지

  * 검토 완료: 초록 계열 배경 + 초록 텍스트
  * 검토 필요: 주황 계열 배경 + 주황 텍스트
  * confidence 높음: 초록 계열
  * confidence 중간: 주황 계열
  * confidence 낮음: 빨강 계열
  * 공개 가능: 초록 계열
  * 비공개: 빨강 계열
  * 정보: 청록 또는 회색 계열 배경 + 정보 텍스트
  * 데이터 없음: 회색 배경 + 회색 텍스트

예:

  * `검토 완료`
  * `검토 필요`
  * `높음`
  * `중간`
  * `낮음`
  * `공개 가능`
  * `비공개`
  * `데이터 없음`

### 필터

필터는 비전문가도 이해하기 쉽게 구성한다.

1차 개발 기본 필터 후보:

  * 전역 검색창
  * 고객사명
  * 산업군
  * 니즈 키워드
  * 소프트웨어
  * review_status
  * confidence
  * sensitivity_level

초기에는 드롭다운을 과도하게 늘리지 않는다. 사용자는 검색창 중심으로 탐색하고, 1차 개발 후 피드백을 받아 필터를 확장한다.

필터 선택 상태는 `raonx-cyan-soft` 배경 또는 `raonx-cyan` 테두리로 표시한다.

### 테이블

  * 고객사명, 니즈 키워드, 소프트웨어명은 좌측 정렬
  * ID, 날짜, 상태값은 중앙 또는 좌측 정렬
  * 긴 텍스트는 기본적으로 2~3줄까지만 보이고 상세 보기에서 전체 표시
  * 헤더는 가능하면 고정
  * 정렬 가능하게 만든다
  * `review_status`, `confidence`, `sensitivity_level`은 텍스트만 쓰지 말고 배지로 표시
  * 너무 많은 컬럼을 한 번에 보여주지 않는다
  * CSV 원본 컬럼명을 그대로 노출하기보다 사용자가 이해하기 쉬운 한글 라벨을 사용한다

## 7. 차트 기준

차트는 많을 필요가 없다. 이 대시보드는 매출 대시보드가 아니라 고객 니즈와 기능 매칭을 빠르게 찾기 위한 지식 검색형 대시보드다.

권장 차트:

| 목적 | 권장 차트 |
|---|---|
| 최근 고객 니즈 현황 | 날짜별 또는 고객사별 리스트 카드 |
| 자주 등장한 니즈 | 키워드 빈도 막대 차트 |
| 검토 필요 매칭 | 상태별 카드 또는 가로 막대 |
| 소프트웨어별 기능 수 | 가로 막대 차트 |
| confidence 분포 | 배지 요약 카드 또는 도넛 후보 |
| 고객사별 대응 전략 수 | 고객사별 리스트 또는 막대 차트 |

차트 원칙:

  * 한 차트에 너무 많은 색을 쓰지 않는다.
  * 핵심 강조색 `raonx-cyan`은 선택 상태 또는 주요 지표에 우선 사용한다.
  * 검토 필요와 비공개는 주황/빨강 계열로 일관되게 표시한다.
  * 범례는 차트 아래 또는 오른쪽에 작게 배치한다.
  * 차트 제목은 단순형, 부제는 해석형으로 작성한다.

예:

  * 제목: `검토 필요 매칭`
  * 부제: `confidence가 낮거나 검토 필요 상태인 항목을 먼저 확인하세요.`

## 8. 화면별 디자인 기준

### 1) Home

Home은 1차 개발의 최우선 화면이다.

첫 화면에서 보여줄 것:

  1. 라온엑스 로고와 서비스명
  2. 마지막 데이터 업데이트 시간
  3. 전체 고객 니즈 수
  4. 전체 매칭 수
  5. 검토 필요 매칭 수
  6. 비공개 항목 수
  7. 최근 고객 니즈
  8. 자주 등장한 니즈 키워드
  9. 고객사별 대응 전략 바로가기

Home은 “전체 현황을 빠르게 파악하고 다음 화면으로 이동하는 출발점”으로 만든다.

### 2) 고객 니즈 검색

  * 전역 검색창을 가장 상단에 둔다.
  * 고객사명, 산업군, 니즈 키워드, 니즈 요약을 함께 검색할 수 있게 한다.
  * 결과는 카드 또는 표로 제공한다.
  * 고객사명과 담당자를 표시한다.
  * `sensitivity_level`과 `dashboard_visible` 상태를 명확히 표시한다.

### 3) 니즈-기능-가치 매칭

  * `need_id`와 `feature_id` 연결 결과를 보여준다.
  * 고객 니즈 → 추천 소프트웨어 → 기능 → 가치 메시지 순서로 읽히게 한다.
  * `review_status`와 `confidence`를 반드시 배지로 표시한다.
  * 검토 필요 항목은 상단 또는 별도 섹션에서 쉽게 찾게 한다.

### 4) 고객사별 대응 전략

  * 고객사명 기준으로 검색한다.
  * `main_needs`, `recommended_software`, `approach_details`, `suggested_questions`, `proposal_message`, `next_action`을 순서대로 보여준다.
  * `review_status`는 검토 필요/검토 완료로 명확히 표시한다.
  * 영업 미팅 전 바로 읽을 수 있는 구조로 만든다.

### 5) 소프트웨어별 가치 맵

  * 소프트웨어명 기준으로 기능 목록을 보여준다.
  * 기능명, 기능 요약, 가치 키워드, 관련 니즈 키워드를 표시한다.
  * `related_need_keywords`는 직접 연결이 아니라 느슨한 키워드 연결임을 화면에 작게 표시한다.
  * 향후 Fegate for Ship, pSeven Desktop, pSeven Enterprise, Nastran, Patran, Apex 외 소프트웨어가 추가될 수 있게 확장 가능한 레이아웃으로 만든다.

## 9. 로그인 화면 기준

로그인 화면은 단순하게 만든다.

구성:

  * 라온엑스 로고
  * 서비스명: `고객 니즈-기능 매칭 대시보드`
  * 아이디 입력
  * 비밀번호 입력
  * 로그인 버튼
  * 오류 메시지
  * 내부 시스템 안내 문구

스타일:

  * 밝은 배경
  * 중앙 정렬 카드
  * 과한 애니메이션 금지
  * 라온엑스 청록색을 로그인 버튼에 사용
  * 로그인 실패는 빨간색으로 명확히 표시하되, 상세 보안 정보는 노출하지 않음
  * 밝은 배경에서는 밝은 바탕용 라온엑스 로고 PNG 사용

안내 문구 예:

> 라온엑스 내부 사용자 전용 시스템입니다.

권한 안내:

  * `admin`: 데이터 수정, 코멘트 관리, 운영 설정 가능
  * `viewer`: 조회 전용
  * 1차 개발에서는 팀원은 기본적으로 `viewer` 권한으로 본다.

## 10. 로고 및 이미지 사용 기준

라온엑스 CI는 대시보드의 신뢰감과 일관성을 위해 적극적으로 사용하되, 과도하게 반복하지 않는다.

### 로고 배치

권장 위치:

  * 로그인 화면 중앙 카드 상단
  * 대시보드 상단 헤더 좌측
  * PDF/인쇄 출력물 상단 좌측
  * 빈 상태 화면 또는 오류 화면의 작은 브랜드 요소

피해야 할 사용:

  * 모든 카드마다 로고 반복
  * 로고 위에 텍스트나 배지 겹치기
  * 로고 색상 임의 변경
  * 로고를 세로/가로로 왜곡
  * X 청록색을 다른 색으로 변경
  * 배경 대비가 낮은 로고 사용

### 로고 파일 사용

  * 밝은 화면: `raonx_logo_light_bg.png`
  * 어두운 헤더/배경: `raonx_logo_dark_bg.png`
  * 단색 워터마크: `raonx_logo_mono.png`

### 여백

  * 로고 주변에는 로고 높이의 최소 0.5배 이상 여백을 둔다.
  * 헤더에서 로고 높이는 28–40px 범위를 우선한다.
  * 로그인 화면에서는 120–180px 폭으로 사용한다.

## 11. PDF/인쇄 출력 기준

인쇄/PDF 출력을 고려한다. 1차 개발 필수는 아니지만, 향후 고객 미팅 준비자료로 출력될 수 있다.

원칙:

  * A4 가로 방향을 우선 고려
  * 배경색은 너무 진하지 않게
  * 카드 그림자는 인쇄 시 최소화
  * 표와 상세 텍스트가 잘리지 않게 페이지 분리 고려
  * 로고, 기준 데이터 업데이트일, 출력일을 상단에 표시
  * PDF에서는 인터랙션 요소를 제거하고 정적인 설명을 추가
  * 비공개/검토 필요/낮은 confidence는 흑백 출력에서도 구분 가능하게 아이콘 또는 라벨을 함께 사용

PDF용 구성 후보:

  1. 표지/제목
  2. 고객사 요약
  3. 주요 니즈
  4. 추천 소프트웨어/기능
  5. 가치 메시지
  6. 다음 미팅 추천 질문
  7. 검토 필요 및 주의사항
  8. source file 및 report date

## 12. 대시보드 출력물 기준

대시보드는 예쁘기보다 미팅 준비가 빨라야 한다.

첫 화면에서 보여줄 것:

  1. 전체 고객 니즈 수
  2. 전체 니즈-기능 매칭 수
  3. 검토 필요 매칭 수
  4. 검토 완료 매칭 수
  5. confidence 낮음 항목 수
  6. 비공개 항목 수
  7. 최근 고객 니즈
  8. 자주 등장한 니즈
  9. 소프트웨어별 기능 수
  10. 데이터 누락/오류/관계 불일치 경고

모든 결론에는 가능한 한 “근거 데이터”와 “다음 확인 항목”이 같이 있어야 한다.

카드 한 줄 요약 예:

  * `검토 필요 매칭이 있습니다. 담당자가 확인 후 상태를 갱신하세요.`
  * `비공개 항목은 고객 미팅 자료로 활용하기 전에 공유 범위를 확인하세요.`
  * `confidence가 낮은 매칭은 제안 메시지로 바로 사용하지 마세요.`
  * `이 고객 니즈와 연결된 추천 기능을 확인하세요.`

## 13. 금지 사항

  * 라온엑스 로고 색상을 임의로 변경하지 않는다.
  * 로고를 찌그러뜨리거나 과도하게 확대하지 않는다.
  * 청록색을 모든 요소에 남발하지 않는다.
  * 빨간색을 과다 사용하지 않는다.
  * 데이터 근거 없는 고객 의도, 구매 가능성, 예산, 도입 의지를 추정하지 않는다.
  * `confidence`가 낮은 매칭을 확정 추천처럼 표시하지 않는다.
  * `review_status = 검토 필요`인 항목을 검토 완료처럼 보이게 하지 않는다.
  * `sensitivity_level = 비공개` 항목을 일반 공개 가능 항목처럼 보이게 하지 않는다.
  * 고객사명과 담당자 정보를 불필요하게 화면에 과도하게 노출하지 않는다.
  * 실제 고객 정보를 예제 화면이나 테스트 이미지에 임의로 사용하지 않는다.
  * 차트를 많이 넣는 대신 핵심 검색과 판단이 안 보이게 만들지 않는다.
  * 과한 애니메이션, 과한 그라데이션, 장식용 그래픽을 사용하지 않는다.
  * 내부 시스템을 외부 공개용 홍보 페이지처럼 보이게 만들지 않는다.
